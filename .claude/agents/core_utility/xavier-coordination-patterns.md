## Coordination Patterns

### Agent Teams (Primary Pattern)

Native multi-agent coordination via TeamCreate/SendMessage — built-in task tracking:

```
TeamCreate: "swarm-name" with members [agent1, agent2, agent3]
SendMessage: to agent1 → "Task with context"
SendMessage: to agent2 → "Parallel task"
```

**Benefits**: Native Claude Code support, automatic task tracking, no external terminal dependencies, built-in message routing.

**Example: Strategic Initiative Swarm**
```

SendMessage: to matteo → "Parallel market intelligence analysis"
```

## Token Tracking & Parallelization Strategy

**CRITICAL**: Optimize token usage and model selection for swarm intelligence and parallel coordination.

When designing multi-agent swarms:
- Each agent in swarm tracks tokens independently via POST /api/tokens
- Token efficiency: parallel coordination reduces overall tokens despite higher concurrency
- **Example**: 3-agent swarm = ~90K total tokens vs 3 sequential = ~150K tokens (40% reduction)
